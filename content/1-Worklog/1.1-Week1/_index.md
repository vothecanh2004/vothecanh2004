### Week 1 Objectives:

* Understand core Cloud Computing concepts: basic definitions, service models (IaaS, PaaS, SaaS), and deployment models (Public, Private, Hybrid Cloud).
* Master AWS Global Infrastructure architecture (Region, AZ, Edge Location, Local Zone) and the Shared Responsibility Model.
* Create an AWS Free Tier account and get familiar with the AWS Management Console interface.
* Configure AWS Budgets to manage costs and receive alerts, ensuring we stay within Free Tier limits.

### Week 1 Key Achievements:

* Mastered Cloud Computing concepts, service models, and clearly distinguished between the 3 deployment models (Public Cloud, Private Cloud, Hybrid Cloud).

* Gained a deep understanding of AWS Global Infrastructure, including the structure and roles of Regions, Availability Zones (AZs), Edge Locations, and Local Zones.

* Fully understood the Shared Responsibility Model between AWS and customers, alongside the general AWS pricing mechanisms.

* **Hands-on Skills:**

* Successfully registered an AWS Free Tier account and became familiar with navigating and operating the AWS Management Console at a basic level.

* Learned how to search, explore, and pin core services (IAM, EC2, Amazon S3) to the Favorites section for easier management in the following weeks.

* Practiced checking the Billing Dashboard, tracking Free Tier usage limits, and successfully configured an AWS Budget to receive alerts, proactively controlling costs to avoid unexpected charges.

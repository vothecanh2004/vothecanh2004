### Week 3 Objectives:

* Understand the Amazon EC2 virtual server service and how to classify EC2 Instance Types (General Purpose, Compute Optimized, Memory Optimized, etc.) based on computing needs.
* Master core operational principles of Security Groups (virtual firewall), Amazon Machine Images (AMIs), and Amazon EBS (block storage volumes).
* Gain hands-on proficiency in launching, managing, and monitoring the lifecycle states of an EC2 Instance directly within the AWS Console.

### Week 3 Key Achievements:

* Mastered the core concepts of Amazon EC2 and clearly differentiated between the overall management service (EC2) and a specific virtual server (EC2 Instance).
* Categorized and understood various EC2 Instance Types (General Purpose, Compute Optimized, Memory Optimized, etc.) to choose the optimal hardware configuration based on specific workload requirements.
* Gained a deep understanding of how Security Groups operate as a stateful firewall at the instance level, and learned to configure Inbound/Outbound rules to secure network traffic.
* Comprehended the distinct roles of Amazon Machine Images (AMI) for providing the baseline operating system/environment, and Amazon Elastic Block Store (EBS) as the persistent block storage volume attached to EC2.
* Successfully completed the hands-on lab: Launched an EC2 Instance, closely monitored its status, and acquired a clear understanding of the transitions within the Instance lifecycle directly on the AWS Console.

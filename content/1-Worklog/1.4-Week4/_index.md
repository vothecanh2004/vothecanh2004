### Week 4 Objectives:

* Understand Amazon S3 object storage mechanics and how to select cost-optimal Storage Classes based on access frequency.
* Master foundational principles of Amazon RDS (SQL relational database) and Amazon DynamoDB (NoSQL non-relational database).
* Gain hands-on proficiency in creating S3 Buckets, configuring Object permissions, and provisioning/connecting to a MySQL instance on Amazon RDS.

### Week 4 Key Achievements:

* Mastered core knowledge of Amazon Simple Storage Service (Amazon S3), object storage mechanics, and foundational concepts including Buckets, Objects, and Keys.
* Clearly differentiated between Amazon S3 Storage Classes and understood how to select the most cost-optimal storage tier based on data access frequency.
* Gained a deep understanding of Amazon RDS (Relational Database Service), its supported database engines (MySQL, PostgreSQL, SQL Server, etc.), and how AWS automates hardware provisioning and backups.
* Explored Amazon DynamoDB alongside the core characteristics of NoSQL databases, and established clear criteria for choosing between Amazon RDS (SQL) and Amazon DynamoDB (NoSQL).
* Successfully completed hands-on labs directly via the AWS Management Console:
  * Created an S3 Bucket and configured flexible security access permissions (Public/Private Access) for the stored Objects inside.
  * Launched a fully functional MySQL instance on Amazon RDS, successfully established a network connection, and verified the accurate operational status of the RDS Instance.
